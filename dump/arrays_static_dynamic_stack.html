Static Array — fixed size, contiguous memory

Size is declared at creation and cannot change. Elements stored in consecutive memory addresses — O(1) random access by index.

Access by index
O(1)
Search (unsorted)
O(n)
Insert / Delete
O(n) — shift needed
Size
Fixed at compile/init time
Memory layout — int arr[6]
Address →
10[0]
20[1]
30[2]
40[3]
50[4]
60[5]

Each cell is 4 bytes (int). arr[3] → base_address + 3×4 — direct jump, no traversal.

Dynamic Array — resizable, amortized O(1) append

Internally a static array with extra capacity. When full, a new array (~2× size) is allocated and elements are copied. Amortized O(1) append.

Append (amortized)
O(1)
Access by index
O(1)
Insert/Delete mid
O(n)
Resize (worst case)
O(n)
— ready —
size: 4  |  capacity: 8  Resize happened!

filled  reserved capacity

Stack — LIFO, array-based implementation

Last In, First Out. Only the top is accessible. Built on a dynamic array — push/pop are O(1). Used in function call frames, undo/redo, expression parsing, DFS.

Push (add to top)
O(1)
Pop (remove top)
O(1)
Peek (view top)
O(1)
Access by index
Not allowed (LIFO)
— ready —
topmost recently pushed
Bottom elements are inaccessible until items above are popped.

// Option 1: Deque (preferred over Stack class) Deque<Integer> stack = new ArrayDeque<>(); stack.push(10); // Push — O(1) stack.push(20); stack.push(30); int top = stack.peek(); // 30, no removal — O(1) int val = stack.pop(); // removes 30 — O(1) System.out.println(stack.isEmpty()); // false // Option 2: Legacy Stack class (avoid in new code) Stack<Integer> s = new Stack<>(); s.push(10); s.peek(); s.pop();
// std::stack (adapter over deque by default) std::stack<int> stack; stack.push(10); // Push — O(1) stack.push(20); stack.push(30); int top = stack.top(); // 30, no removal — O(1) stack.pop(); // removes top — O(1) (returns void!) if (stack.empty()) std::cout << "empty"; std::cout << stack.size(); // Use vector as underlying container std::stack<int, std::vector<int>> vs; vs.push(10);
